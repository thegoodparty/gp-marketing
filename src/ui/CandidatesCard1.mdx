# Candidates Card 1

<Description of={CandidatesCard1Stories.Default} />

## Overview

CandidatesCard1 is a standard candidate card component used in the CandidatesBlock grid. It displays candidate information with an avatar (or initials fallback), name, party affiliation, and a "View profile" link. This card variant is used for non-GoodParty candidates.

## Usage

Use this component when:
- Displaying standard candidates in a CandidatesBlock grid
- Showing candidates who are not GoodParty-empowered
- Creating a consistent card layout for candidate listings

### Prop Details

#### `name` (required)

The candidate's full name.

**Type:** `string`

#### `partyAffiliation` (required)

The candidate's party affiliation or role title.

**Type:** `string`

#### `href` (required)

The URL path to the candidate's profile page.

**Type:** `string`

#### `avatar` (optional)

The candidate's profile image. Can be a SanityImage object or a string URL.

**Type:** `SanityImage | string`

#### `className` (optional)

Additional CSS classes to apply to the component.

**Type:** `string`

## Avatar Fallback

When no `avatar` is provided, the component automatically generates initials from the candidate's name:
- Takes the first letter of each word in the name
- Converts to uppercase
- Displays up to 2 characters
- Shows in a gray circular background

**Example:** "Jane Smith" → "JS", "Maria Hernandez-Rodriguez" → "MH"

## Layout

The card uses a responsive flex layout:
- Mobile: Vertical stack (avatar, content, link)
- Desktop: Horizontal layout with avatar on left, content in middle, link on right
- Includes hover effects (shadow on hover)

## Accessibility

- Uses semantic `<article>` element
- Avatar images include proper alt text
- "View profile" link is keyboard accessible
- Arrow icon provides visual indication of link
