# Elections Position Content Block

<Description of={ElectionsPositionContentBlockStories.Default} />

## Overview

The ElectionsPositionContentBlock component displays a two-column layout with a white card on the left containing headline, subhead, body copy, and a primary CTA button. The right side features a top headline, a three-column grid of subhead/body copy pairs, and optional bottom headline/body copy sections separated by a horizontal line.

## Usage

Use this component when:
- Displaying election position information with a prominent call-to-action
- Showing structured content in a grid layout
- Creating content sections with multiple related items
- Presenting information that benefits from a card-based left column and grid-based right column

## Design Features

- **Left Card**: White card with rounded corners containing headline, subhead, body copy, and primary CTA
- **Right Content**: Flexible content area with top headline, grid items, and bottom items
- **Three-Column Grid**: Responsive grid that displays subhead/body copy pairs
- **Horizontal Separator**: Visual separator between grid items and bottom items
- **Responsive Layout**: Two-column layout on desktop, stacks on mobile

## Props

<Controls of={ElectionsPositionContentBlockStories.Default} />

### Prop Details

#### `card` (optional)

Left side card configuration with headline, subhead, body copy, and primary CTA.

**Type:** `ElectionsPositionContentBlockCardProps`

Card properties:
- `headline` (optional) - Medium heading text (heading-md)
- `subhead` (optional) - Overline text (overline)
- `bodyCopy` (optional) - Body text content, can be a single ReactNode or array of ReactNodes
- `primaryCTA` (optional) - Primary call-to-action button configuration

#### `topHeadline` (optional)

Headline text displayed at the top of the right content area.

**Type:** `string`

Uses `subtitle-1` text style.

#### `gridItems` (optional)

Array of items to display in the three-column grid on the right side.

**Type:** `ElectionsPositionContentBlockGridItem[]`

Each grid item contains:
- `subhead` (optional) - Overline text (overline)
- `bodyCopy` (optional) - Body text content (body-2)

#### `bottomItems` (optional)

Array of items to display below the grid section, separated by a horizontal line.

**Type:** `ElectionsPositionContentBlockBottomItem[]`

Each bottom item contains:
- `headline` (optional) - Subtitle text (subtitle-1)
- `bodyCopy` (optional) - Body text content (body-2)

#### `backgroundColor` (optional)

Background color variant for the block.

**Type:** `'cream' | 'midnight'`

**Default:** `'cream'`

When set to `'midnight'`:
- Background becomes `bg-midnight-900`
- Right content text becomes white
- Card remains white
- Separator border becomes `border-neutral-700`

#### `className` (optional)

Additional CSS classes to apply to the component.

**Type:** `string`

## Layout

The component uses a responsive grid layout:
- **Mobile**: Single column (card appears above content)
- **Desktop** (`lg:`): Two-column grid
  - Left: Card with fixed width (`1fr`)
  - Right: Content area (`1fr`)
  - Gap: `gap-20` between columns

### Card Layout

The left card features:
- White background (`bg-white`)
- Rounded corners (`rounded-lg`)
- Responsive padding: `p-4` (mobile), `md:p-6` (tablet), `lg:p-8` (desktop)
- Vertical flex layout with gap: `gap-3` (mobile), `gap-4` (desktop)
- Content section with `gap-4` between elements

### Grid Layout

The right side grid:
- **Mobile**: Single column
- **Tablet** (`md:`): Two-column grid
- **Desktop** (`lg:`): Three-column grid
- Gap: `gap-6` (mobile), `gap-8` (desktop)
- Each grid item has vertical flex layout with `gap-3`

### Separator

The horizontal separator:
- Only displays when both `gridItems` and `bottomItems` are present
- Border: `border-t border-neutral-200` (cream background)
- Border: `border-t border-neutral-700` (midnight background)

## Typography

- **Card headline**: `heading-md` - Medium, bold heading
- **Card subhead / Grid subhead**: `overline` - Overline text
- **Top headline / Bottom headline**: `subtitle-1` - Subtitle text
- **Body copy**: `body-1` (card), `body-2` (grid and bottom items)

## Button Styling

The primary CTA button uses `resolveButtonStyleType` to automatically adjust its style based on the background color:
- **Cream background**: Uses the specified button style type
- **Midnight background**: Automatically adjusts for dark background visibility

Button types supported:
- `internal` - Internal navigation link
- `external` - External link
- `anchor` - Anchor link
- `download` - Download link
- `contact` - Contact link
- `login` - Login action
- `signup` - Signup action
- `button` - Button with onClick handler

## Rich Text Support

Body copy fields support rich text content via the `isValidRichText` helper:
- Validates content before rendering
- Supports ReactNode content types
- Handles empty or null content gracefully

## Content Structure Examples

### Full Content Block

```tsx
<ElectionsPositionContentBlock
  card={{
    headline: 'Headline',
    subhead: 'Subhead',
    bodyCopy: ['Body copy', 'Body copy'],
    primaryCTA: {
      buttonType: 'internal',
      href: '/action',
      label: 'Primary CTA',
    },
  }}
  topHeadline="Headline"
  gridItems={[
    { subhead: 'Subhead', bodyCopy: 'Body copy' },
    { subhead: 'Subhead', bodyCopy: 'Body copy' },
    { subhead: 'Subhead', bodyCopy: 'Body copy' },
  ]}
  bottomItems={[
    { headline: 'Headline', bodyCopy: 'Body copy' },
    { headline: 'Headline', bodyCopy: 'Body copy' },
  ]}
/>
```
