# Candidates Card

<Description of={CandidatesCardStories.Standard} />

## Overview

The CandidatesCard component is a unified candidate card that displays candidate information with responsive layouts for mobile and desktop. It supports both standard candidates and GoodParty-empowered candidates through conditional styling and content rendering. The component automatically handles avatar fallback with initials and includes proper spacing and alignment per Figma specifications.

## Usage

Use this component when:
- Displaying candidate information in CandidatesBlock grids
- Showing both standard and GoodParty-empowered candidates
- Creating responsive candidate cards that match Figma design specifications
- Building candidate listings with consistent styling

### Prop Details

#### `name` (required)

The candidate's full name.

**Type:** `string`

#### `partyAffiliation` (required)

The candidate's party affiliation or role title.

**Type:** `string`

#### `href` (required)

The URL path to the candidate's profile page.

**Type:** `string`

#### `avatar` (optional)

The candidate's profile image. Can be a SanityImage object or a string URL.

**Type:** `SanityImage | string`

#### `isGoodPartyCandidate` (optional)

Flag indicating this is a GoodParty-empowered candidate. When `true`, the card displays:
- Yellow border (`border-bright-yellow-600`)
- Yellow hover background (`hover:bg-bright-yellow-50`)
- GoodParty logo badge overlay on avatar
- "Empowered by goodparty.org" text in gray

**Type:** `boolean`

**Default:** `false`

#### `className` (optional)

Additional CSS classes to apply to the component.

**Type:** `string`

#### `_key` (optional)

Internal key used by CandidatesBlock for React list rendering.

**Type:** `string`

## Layout Specifications

### Mobile (below md breakpoint)
- Vertical layout (`flex-col`)
- Padding: `p-6` (24px)
- Gap between elements: `gap-2` (8px)
- Avatar size: `size-20` (80px)
- Name: Single line (`whitespace-nowrap`)
- "View profile" link: Right-aligned at bottom

### Desktop (md breakpoint and above)
- Horizontal layout (`md:flex-row`)
- Padding: `p-6` (24px)
- Gap between avatar and content: `md:gap-6` (24px)
- Alignment: Top-aligned (`md:items-start`)
- Avatar size: `md:size-24` (96px)
- Name: Allows wrapping (`md:whitespace-normal`) for long names
- "View profile" link: Right-aligned, vertically centered

## Styling Variants

### Standard Candidate (`isGoodPartyCandidate` false/undefined)
- Border: `border-black/8` (gray)
- Hover: Shadow only (`hover:shadow-md`)
- No badge overlay
- No "Empowered by" text

### GoodParty Candidate (`isGoodPartyCandidate` true)
- Border: `border-bright-yellow-600` (yellow)
- Hover: Shadow and yellow background (`hover:shadow-md hover:bg-bright-yellow-50`)
- Badge: GoodParty logo positioned `absolute -bottom-0.5 -right-0.5` on avatar
- "Empowered by goodparty.org": Displayed in `text-neutral-500` (gray), `caption` styleType

## Avatar Fallback

When no `avatar` is provided, the component automatically generates initials from the candidate's name:
- Takes the first letter of each word in the name
- Converts to uppercase
- Displays up to 2 characters
- Shows in a gray circular background (`bg-gray-200`)

**Example:** "Jane Smith" → "JS", "Maria Hernandez-Rodriguez" → "MH"

## Badge Overlay

For GoodParty candidates, the logo badge:
- Size: 50px × 35px (`w-[50px] h-[35px]`)
- Position: `absolute -bottom-0.5 -right-0.5` (extends slightly outside avatar boundary)
- Contains: GoodParty Logo component (`width={100} height={75}`)

## Accessibility

- Uses semantic `<article>` element
- Avatar images include proper alt text via Avatar component
- "View profile" link is keyboard accessible
- Arrow icon provides visual indication of external navigation
- Attribution text provides context about GoodParty empowerment
