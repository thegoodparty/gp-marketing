# Featured Cities Block

<Description of={FeaturedCitiesBlockStories.Default} />

## Overview

The FeaturedCitiesBlock component displays a grid of LocationCard components showcasing featured cities with their state abbreviations, open elections counts, and links. It includes an optional header and returns null if no location cards are provided.

## Usage

Use this component when:
- Highlighting featured cities on state or county pages
- Displaying cities with open elections
- Creating a visual grid of location cards
- Showcasing key cities in a region

### Prop Details

#### `locationCards` (required)

Array of LocationCard objects to display in the grid.

**Type:** `LocationCardProps[]`

Each LocationCard contains:
- `cityName` (required) - Name of the city
- `stateAbbreviation` (required) - Two-letter state abbreviation
- `openElectionsCount` (optional) - Number of open elections
- `href` (required) - Link to city elections page

#### `header` (optional)

HeaderBlock configuration for the section title and description.

**Type:** `HeaderBlockProps`

#### `backgroundColor` (optional)

Background color variant for the block.

**Type:** `'cream' | 'midnight'`

**Default:** `'cream'`

#### `className` (optional)

Additional CSS classes to apply to the component.

**Type:** `string`

## Conditional Rendering

The component returns `null` if:
- `locationCards` is not provided
- `locationCards` is an empty array

This prevents rendering empty sections.

## Grid Layout

The location cards are displayed in a responsive grid:
- **Mobile**: Single column
- **Tablet** (`sm:`): 2 columns (`sm:grid-cols-2`)
- **Desktop** (`md:`): 2 columns (`md:gap-6`)
- **Large Desktop** (`lg:`): 3 columns (`lg:grid-cols-3`)
- Gap: `gap-4` on mobile, `gap-6` on desktop

## LocationCard Features

Each LocationCard displays:
- State icon (from `/icons/states/{abbreviation}.svg`)
- City name and state abbreviation
- Open elections count (if provided)
- "View elections" link with arrow icon
- Hover effects (background color change, arrow translation)

## Accessibility

- Uses semantic `<article>` element
- Grid uses semantic `<ul>` and `<li>` structure
- LocationCard links are keyboard accessible
- State icons include proper alt text
- HeaderBlock provides proper heading hierarchy
